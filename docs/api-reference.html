API Reference

Overview

The ABET Assessment Application provides a RESTful API for integrating with external systems. The API uses JSON for request and response payloads.

Authentication

All API requests require authentication using a bearer token. Include your token in the Authorization header:

Authorization: Bearer YOUR_API_TOKEN

To obtain an API token, contact your system administrator.

Base URL

https://api.abet-assessment.edu/v1

Endpoints

Assessments

GET /assessments

Retrieve a list of assessments

Query Parameters:

  • program_id (optional): Filter by program
  • year (optional): Filter by academic year
  • limit (optional): Number of results to return (default: 20)
  • offset (optional): Pagination offset (default: 0)

Response:

{
  "data": [
    {
      "id": "1",
      "program_name": "Computer Science",
      "year": 2026,
      "status": "active",
      "created_at": "2026-01-15"
    }
  ],
  "total": 1,
  "limit": 20,
  "offset": 0
}

GET /assessments/{id}

Retrieve a specific assessment

Parameters:

  • id (required): Assessment ID

Response:

{
  "id": "1",
  "program_name": "Computer Science",
  "year": 2026,
  "description": "Assessment data for 2026",
  "status": "active",
  "created_at": "2026-01-15",
  "updated_at": "2026-03-01"
}

Program Outcomes

GET /assessments/{id}/outcomes

Retrieve program outcomes for an assessment

Response:

{
  "data": [
    {
      "id": "101",
      "assessment_id": "1",
      "description": "Students will understand core algorithms",
      "type": "knowledge",
      "created_at": "2026-01-15"
    }
  ]
}

POST /assessments/{id}/outcomes

Create a new program outcome

Request Body:

{
  "description": "Students will apply design patterns",
  "type": "skill"
}

Response:

{
  "id": "102",
  "assessment_id": "1",
  "description": "Students will apply design patterns",
  "type": "skill",
  "created_at": "2026-03-01"
}

Student Data

POST /assessments/{id}/student-data

Submit student performance data

Request Body:

{
  "student_id": "STU001",
  "outcome_id": "101",
  "performance_rating": 4,
  "notes": "Demonstrated excellent understanding"
}

Response:

{
  "id": "DATA001",
  "assessment_id": "1",
  "student_id": "STU001",
  "outcome_id": "101",
  "performance_rating": 4,
  "notes": "Demonstrated excellent understanding",
  "created_at": "2026-03-01"
}

Reports

GET /assessments/{id}/report

Generate a report for an assessment

Query Parameters:

  • format: csv, json, or pdf (default: json)

Response: Report data in requested format

Error Handling

The API uses standard HTTP status codes:

  • 200 OK: Successful request
  • 201 Created: Resource successfully created
  • 400 Bad Request: Invalid request parameters
  • 401 Unauthorized: Missing or invalid authentication
  • 404 Not Found: Resource not found
  • 500 Internal Server Error: Server error

Error responses include a message and error code:

{
  "error": {
    "code": "INVALID_OUTCOME_ID",
    "message": "The specified outcome does not exist"
  }
}

Rate Limiting

API requests are rate-limited to 1000 requests per hour per token. The response headers include:

  • X-RateLimit-Limit: Maximum requests per hour
  • X-RateLimit-Remaining: Remaining requests in current hour
  • X-RateLimit-Reset: Unix timestamp of rate limit reset
