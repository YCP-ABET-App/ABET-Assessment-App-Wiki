Deployment Guide

Overview

This guide covers the deployment process for the ABET Assessment Application. The application can be deployed to various environments including development, staging, and production.

System Requirements

Server Requirements

  • Operating System: Linux (Ubuntu 20.04+), Windows Server 2019+, or macOS
  • CPU: Minimum 2 cores (4 cores recommended for production)
  • RAM: Minimum 4GB (8GB recommended for production)
  • Storage: Minimum 10GB free space

Software Requirements

  • Node.js v16.x or higher
  • NPM v8.x or higher
  • PostgreSQL 12.x or higher (for database)
  • Redis 6.x or higher (for caching)
  • Docker (optional, for containerized deployment)

Network Requirements

  • Open port 443 for HTTPS
  • SSL/TLS certificate (required for production)
  • Firewall rules configured for database and Redis access

Installation Steps

1. Prepare the Server

  1. Update system packages: 
    sudo apt update && sudo apt upgrade
  2. Install Node.js and NPM: 
    curl -fsSL https://deb.nodesource.com/setup_16.x | sudo -E bash -
sudo apt-get install -y nodejs
  3. Install PostgreSQL: 
    sudo apt install postgresql postgresql-contrib
  4. Install Redis: 
    sudo apt install redis-server

2. Clone the Repository

git clone https://github.com/your-org/abet-assessment-app.git
cd abet-assessment-app

3. Install Dependencies

npm install

4. Configure Environment

Create a .env file with the following configuration:

NODE_ENV=production
PORT=3000
DATABASE_URL=postgresql://user:password@localhost:5432/abet_db
REDIS_URL=redis://localhost:6379
JWT_SECRET=your-secret-key
ADMIN_EMAIL=admin@institution.edu

5. Initialize the Database

npm run db:migrate
npm run db:seed

6. Build for Production

npm run build

7. Start the Application

npm start

Docker Deployment

Using Docker Compose

The application includes a docker-compose.yml file for easy deployment:

docker-compose up -d

Building a Custom Docker Image

docker build -t abet-assessment:latest .
docker run -d -p 443:3000 \
  -e DATABASE_URL=postgresql://user:pass@db:5432/abet \
  -e REDIS_URL=redis://redis:6379 \
  abet-assessment:latest

SSL/TLS Configuration

Using Let's Encrypt with Nginx

  1. Install Certbot: 
    sudo apt install certbot python3-certbot-nginx
  2. Obtain certificate: 
    sudo certbot certonly --nginx -d abet-assessment.institution.edu
  3. Configure Nginx to use the certificate
  4. Set up auto-renewal: 
    sudo systemctl enable certbot.timer

Database Backup

Backup Procedure

pg_dump -U postgres abet_db > backup_$(date +%Y%m%d).sql

Restore Procedure

psql -U postgres abet_db < backup_YYYYMMDD.sql

Automated Backups

Add to crontab to run daily backups at 2 AM:

0 2 * * * pg_dump -U postgres abet_db > /backups/backup_$(date +\%Y\%m\%d).sql

Monitoring and Maintenance

Application Logs

Monitor application logs using:

tail -f logs/app.log

Health Check

Monitor application health at:

GET /health

Database Maintenance

  • Run VACUUM regularly
  • Monitor disk space
  • Archive old logs
  • Check connection limits

Performance Monitoring

Use tools like:

  • New Relic
  • DataDog
  • Prometheus
  • Grafana

Troubleshooting

Common Issues

  • Port Already in Use: Change PORT in .env or kill the process using port 3000
  • Database Connection Failed: Verify DATABASE_URL and ensure PostgreSQL is running
  • Out of Memory: Increase server RAM or configure Node.js memory limits
  • Slow Performance: Check database indexes and Redis cache status

Log Files

Check logs at:

  • Application logs: logs/app.log
  • Error logs: logs/error.log
  • Database logs: /var/log/postgresql/
